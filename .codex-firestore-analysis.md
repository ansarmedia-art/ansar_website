# Firestore access analysis (working note)

- Target: `ansar-english-school/(default)`, Standard edition, `asia-south2`.
- Authentication: Firebase Auth; admin authorization is derived from custom admin claim, a fixed email allowlist, or an existing `admins/{email}` document.
- Existing public content reads: pages, updates/news/events, gallery, navigation, settings, sports, achievements, leadership, notices and related school content.
- Existing admin writes: the matching public-content collections plus `adminTrash`.
- User tracking: authenticated users write only their own `users/{uid}` document; admin reads the user activity collection ordered by `lastLogin`.
- New private path: `electionAdminConfig/primary`.
- New document schema: `analyticsKey` string (20-128 restricted characters), `updatedAt` timestamp equal to request time, `updatedBy` verified admin email string (1-200 characters).
- New client operations: authenticated admin `getDoc` and `setDoc(..., {merge:true})` on the fixed document only. No list query, public read, delete, or official access is required.
- Officials use the Apps Script keyed analytics URL and never read this Firestore document.

## Devil's-advocate audit of the new path

- Public list/get: denied because every read requires `isVerifiedAdmin()` and the fixed `primary` ID.
- Unauthenticated or ordinary authenticated user read/write: denied; authority comes only from Firebase Auth plus the existing protected admin sources.
- Create/update bypass: denied; both operations call the same strict validator.
- Schema pollution and required-field omission: denied by `hasAll` plus `hasOnly`.
- Type juggling and resource exhaustion: denied by timestamp/string checks and 20-128/200 character limits.
- Timestamp manipulation: denied because `updatedAt` must equal `request.time`.
- Identity spoofing: denied because `updatedBy` must equal the verified Firebase Auth token email.
- Delete/replay/orphan/query attacks: delete is always denied; there are no counters or subcollections; the client uses a fixed document get.
- Application compatibility: authorized admins perform exactly `getDoc` and validated `setDoc` on `electionAdminConfig/primary`; officials do not use Firestore.
